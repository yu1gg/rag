## 三、模块接口详细定义

### 3.1 数据采集模块（`src/data/collector.py`） ✅ 已实现

定义统一数据结构 `Document` 和 `QAPair`，包含四个采集器：
- `ArxivCollector` — arXiv论文采集，支持 arxiv 包和 API 降级两种模式
- `NewsCollector` — 新闻采集，优先在线获取（HackerNews API），不足时模板补充
- `CourseMaterialCollector` — 课程资料采集，读取本地 .md/.txt 文件，无文件时生成默认10章课程内容
- `QAPairCollector` — QA对采集，优先加载已有JSON，不足时基于10个知识主题的20+模板自动生成

辅助函数：`save_documents()`、`save_qa_pairs()`、`load_documents()` 提供 JSON 序列化/反序列化。

### 3.1.1 补充文档生成器（`src/data/supplement.py`） ✅ 已实现

`generate_supplement_docs(target_chars, existing_count)` — 内置20篇深度技术文章（每篇2000-4000字），当采集总量不足10万字时自动循环补充，确保达标。

### 3.2 文本清洗模块（`src/data/cleaner.py`） ✅ 已实现

- `TextCleaner.clean(text)` — 四步清洗流水线：BeautifulSoup去HTML → 全角转半角 → 空白符归一化 → 控制字符移除
- `TextCleaner.clean_documents(docs)` — 批量清洗，自动过滤清洗后不足20字的文档
- `Deduplicator.deduplicate(docs)` — 基于文本指纹去重（前后200字+长度签名），阈值0.9
- `Deduplicator.filter_short(docs, min_length)` — 过滤字符数不足的短文档

### 3.3 文本分块模块（`src/data/chunker.py`） ✅ 已实现

`Chunker(chunk_size=512, overlap=128)` 滑动窗口分块器：
- `chunk(doc)` — 单文档分块，先按 `\n\n` 切段落，再在段落内滑动窗口合并
- `chunk_all(docs)` — 批量分块并全局统一编号
- 支持 tiktoken 精确计数，失败时回退字符估算（1.5字/token）
- overlap 策略：从上一窗口末尾取重叠字符，调整到最近的段落边界

### 3.4 嵌入模块（`src/embedding/embedder.py`） ✅ 已实现

`Embedder(model_name, device)` — 基于 sentence-transformers 的 BGE-M3 向量化器：
- `encode(texts, batch_size)` — 批量文本→1024维 L2 归一化向量，大文本自动显示进度条
- `encode_query(query)` — 单条查询向量化，延迟加载模型（首次使用时加载）

### 3.5 向量存储模块（`src/embedding/vector_store.py`） ✅ 已实现

`VectorStore(dim=1024)` — 基于 FAISS IndexFlatIP 的向量索引管理：
- `add(chunk_ids, vectors, contents, doc_ids)` — 批量添加向量及元数据
- `search(query_vec, k)` — 余弦相似度检索，返回 (internal_id, score) 列表
- `save(index_path, meta_path)` / `load(index_path, meta_path)` — 索引与元数据持久化

### 3.6 检索模块（`src/retrieval/retriever.py`） ✅ 已实现

`Retriever(embedder, vector_store)` — 组合嵌入器和向量存储：
- `retrieve(query, k=5)` — Query向量化→FAISS检索→封装为 `SearchResult` 列表
- `SearchResult` 数据类：chunk_id, doc_id, content, score, doc_title, source

### 3.7 重排序模块（`src/retrieval/reranker.py`） ✅ 已实现

`Reranker(model_name)` — Cross-Encoder 重排序器，双后端策略：
- FlagEmbedding 优先（bge-reranker-v2-m3），失败自动降级为 sentence-transformers CrossEncoder
- `rerank(query, results)` — 对检索结果逐对打分，按分数降序返回

### 3.8 提示词模板模块（`src/generation/prompts.py`） ✅ 已实现

`PromptTemplate` 类，管理三套提示词模板（QA_TEMPLATE / SUMMARY_TEMPLATE / KNOWLEDGE_QA_TEMPLATE），提供两个核心方法：
- `format_context(results)` — 将 SearchResult 列表格式化为带 `[来源N]` 编号的上下文字符串
- `format_prompt(mode, question, context)` — 根据模式（qa/summary/knowledge_qa）选择模板并填充

### 3.9 LLM 客户端模块（`src/generation/llm_client.py`） ✅ 已实现

`LLMClient` 类，基于 `openai` 库的 OpenAI 兼容接口封装，支持 DeepSeek / MiMo / Qwen 切换。提供三个核心方法：
- `chat(messages, temperature, max_tokens)` — Chat Completion 调用，指数退避重试（最多3次）
- `answer(prompt)` — 单轮问答快捷方法
- `summarize_long_text(text)` — 长文本分段摘要+合并二次归纳

### 3.10 自动评估模块（`src/evaluation/auto_eval.py`） ✅ 已实现

`AutoEvaluator` 类，四个评估方法：
- `evaluate_rouge_l(predictions, references)` — ROUGE-L Precision/Recall/F1
- `evaluate_bleu(predictions, references)` — 语料级 BLEU + BP + 1-gram
- `evaluate_bert_score(predictions, references, lang)` — BERTScore 语义相似度（基于 bert-base-chinese）
- `evaluate_hit_at_k(retrieved, relevant, k)` — Hit@K 检索命中率

`format_report(metrics)` — 将指标字典格式化为可读报告文本。

### 3.11 人工评估模块（`src/evaluation/human_eval.py`） ✅ 已实现

`HumanScore` 数据类：question_id, question, answer, correctness/completeness/fluency (1-5), case_type, analysis。

`HumanEvaluator(output_path)` — JSON 持久化存储：
- `record(score)` / `record_batch(questions, answers, scores_matrix)` — 记录评分
- `statistics()` — 统计平均分 + 按案例类型分组
- `generate_report()` — 生成完整可读报告

### 3.12 UI 模块（`src/ui/app.py`） ✅ 已实现

`RAGApp(config)` — 初始化时自动加载 FAISS 索引、嵌入器、检索器、重排序器和 LLM 客户端。三个处理函数：
- `qa_handler(question, top_k, temperature)` — 检索→重排→QA模板→LLM 生成 + 参考片段展示
- `summary_handler(text, temperature)` — 直接调用摘要模板（长文本自动分段）
- `knowledge_qa_handler(question, top_k, temperature)` — 知识问答模板 + 引用来源标注

`build_ui()` — 使用 `gr.Blocks` + `gr.Tabs` 构建三 Tab 界面，每个 Tab 包含输入区、参数滑块、结果区和参考文档展示区。

## 五、主入口设计（`main.py`） ✅ 已实现

基于 `argparse` 的 CLI 入口，通过 `-c` 参数指定配置文件路径，分发到各子命令处理函数。

| 子命令 | 功能 | 依赖阶段 |
|--------|------|---------|
| `collect` | 数据采集 | - |
| `clean` | 文本清洗去重 | collect |
| `chunk` | 文本分块 | clean |
| `index` | 向量化 + FAISS 索引构建 | chunk |
| `serve` | 启动 Gradio Web 服务 | index |
| `eval` | 运行自动评估 + 人工评估记录 | serve（需先有预测结果） |
| `pipeline` | 一键全流程 | - |

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| DeepSeek API 不可用 | 生成层失效 | 切换 MiMo API，仅需改 base_url 和 api_key |
| BGE-M3 模型下载慢 | 嵌入层阻塞 | 使用 HuggingFace 镜像，或备选 text2vec-large-chinese |
| FAISS 索引内存不足 | 检索崩溃 | 切换为 `IndexIVFFlat`（近似搜索），降低内存占用 |
| 采集数据量不达标 | 评估样本不足 | 使用模板批量生成 QA 对，扩充公开数据源 |
| Gradio 版本兼容问题 | UI 异常 | 锁定 gradio>=4.0，用 `gr.Blocks` 而非旧版 `gr.Interface` |
