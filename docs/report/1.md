> 实践课题：**基于BERT预训练模型的外卖评论情感分类研究与实现**  
> 指导教师：**王伟**

## 一、选题背景
随着外卖平台的发展，用户评论数量不断增加。评论文本中包含用户对菜品质量、配送速度、包装情况和服务态度等方面的反馈。对这些评论进行情感分类，可以帮助快速判断用户评价倾向，具有一定的实际应用价值。

传统文本分类方法通常依赖词频、TF-IDF等人工特征，难以充分理解评论中的上下文语义。BERT是基于Transformer的预训练语言模型，能够提取更丰富的文本语义特征。因此，本课题拟采用BERT模型完成中文外卖评论的正负情感分类。

## 二、设计目标
本课题计划实现一个中文外卖评论情感分类模型。模型输入一条中文评论文本，输出该评论属于正向评价或负向评价。

主要目标如下：
1. 完成中文外卖评论数据集的整理与预处理。
2. 使用中文BERT预训练模型提取评论文本语义特征。
3. 构建正向、负向二分类模型。
4. 设置TF-IDF+逻辑回归作为传统方法对比。
5. 使用Accuracy、Precision、Recall、F1-score等指标评价模型效果。

## 三、数据收集
采用公开中文外卖评论情感分类数据集作为实验数据。数据集中每条样本主要包括评论文本和情感标签，标签分为正向评论和负向评论。

数据示例如下：

| 评论文本 | 标签 |
|----------|------|
| 配送很快，饭菜还是热的，味道也不错 | 正向 |
| 等了一个多小时，菜都凉了，体验很差 | 负向 |

数据集计划按照训练集、验证集和测试集进行划分，例如训练集占70%，验证集占15%，测试集占15%。若正负样本数量差距较大，则采用分层划分方式，保证各部分数据类别比例基本一致。

## 四、数据预处理
首先对原始评论数据进行清洗，删除空评论、重复评论、标签缺失和格式错误的数据，去除多余空格、异常字符等无意义内容。对于评论中的常见标点、语气词等内容，不进行过度删除，以免丢失情感信息。

然后将情感标签转换为数字形式，例如正向评论标记为1，负向评论标记为0。对于BERT模型，使用`bert-base-chinese`对应的中文Tokenizer对文本进行编码，生成`input_ids`和`attention_mask`等输入。设置统一最大文本长度，如128，短文本进行填充，长文本进行截断。

## 五、模型设计
本课题拟采用“BERT预训练模型+分类层”的结构完成评论情感分类。整体流程如下：

```
外卖评论文本 → BERT Tokenizer编码 → BERT模型 →  CLS 向量 → 全连接分类层 → Softmax输出
```

BERT模型用于提取评论文本的上下文语义特征。实验中选取` CLS `位置的输出向量作为整条评论的语义表示，再输入全连接分类层，输出正向和负向两个类别的概率。训练时计划采用交叉熵损失函数和AdamW优化器。

为了体现BERT方法的效果，计划设置TF-IDF+逻辑回归作为对比方法。该方法实现简单、训练速度快，可以作为基线模型，与BERT模型的分类效果进行比较。

## 六、实验方案
初步训练参数设计如下：

| 参数 | 设置 |
|------|------|
| 预训练模型 | bert-base-chinese |
| 最大文本长度 | 128 |
| Batch Size | 8或16 |
| 学习率 | 2e-5 |
| 训练轮数 | 3到5 |
| 优化器 | AdamW |
| 损失函数 | CrossEntropyLoss |

实验步骤计划如下：
1. 收集并整理中文外卖评论数据集。
2. 完成数据清洗、标签转换和数据集划分。
3. 构建TF-IDF+逻辑回归基线模型。
4. 加载中文BERT预训练模型和Tokenizer。
5. 微调BERT文本分类模型。
6. 在测试集上计算评价指标。
7. 对比两种模型结果，并分析典型预测样例。

## 七、评价方法
模型训练完成后，计划使用Accuracy、Precision、Recall、F1-score等指标进行评估。

| 指标 | 说明 |
|------|------|
| Accuracy | 衡量整体分类正确率 |
| Precision | 衡量预测为某类样本时的准确程度 |
| Recall | 衡量某类真实样本被识别出的比例 |
| F1-score | 综合反映Precision和Recall |
| 混淆矩阵 | 分析正负评论的具体误分类情况 |

同时计划绘制混淆矩阵，观察正向评论和负向评论的识别情况，并选取部分预测正确和预测错误的样例进行分析。

## 九、可行性分析
本课题数据来源较容易获取，模型方案成熟，实验流程清晰。相比目标检测、图像分割和图像生成任务，文本分类对硬件要求更低，在普通实验环境下即可完成。若BERT训练速度较慢，可适当减少训练数据规模、降低Batch Size或减少训练轮数，保证课程设计能够按时完成。
